# Napkin Skill

## What
A living document (.claude/napkin.md) that records mistakes, gotchas, and surprising behaviors encountered during development.

## When to Write
- After any unexpected error or bug
- When a tool behaves differently than expected
- When an approach fails and you find a better one
- When external service has undocumented behavior

## Format
```
### [Category]
- [Date] Brief description of what went wrong and the fix
```

## When to Read
- Before starting any new task (check for relevant past mistakes)
- Before using a tool or service for the first time in a session
- When debugging — maybe this was hit before
